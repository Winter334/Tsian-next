# Tsian Agent Notes

## 主目标

当前目标是 `推进 Tsian 项目实现`。

## 实现原则

- 优先推进当前用户正在做的实现任务，不要把精力转移到补规则、补说明、补形式化文档上。
- 先看代码和 `docs/` 中当前对应主题的文档；如果仍然不明确，先问用户，不要自行补设计。
- 每次只做当前阶段最小必要实现，不顺手扩到下一层系统。
- 每一步都应尽量有可见效果、可验收结果。

## 原型期边界

- 当前是快速原型期，禁止过度设计和提前优化。
- 禁止擅自补迁移、兼容层、历史数据保留逻辑，除非用户明确要求。
- 禁止因为局部实现便利，就提前引入通用框架、通用 patch 系统、通用资源层。
- 浏览器本地存储、IndexedDB、Dexie 数据在当前阶段默认允许破坏性调整。
- 如果某个问题更适合“清本地数据重建”而不是“补兼容”，原型期默认选前者。

## 何时停下来问用户

- 文档和代码没有覆盖当前实现细节，且存在多种合理方案。
- 代码变更会引入迁移、兼容层、数据保留逻辑。
- 需要新增较重的抽象层、统一框架或提前工程化能力。
- 当前代码与已确认文档口径明显冲突，且无法通过局部修正解决。

## 验证

- 修改后优先做与当前改动最相关的最小验证，不要无关扩测。
- 当前仓库可直接使用的命令：
  - `npm run dev:web`
  - `npm run dev:server`
  - `npm run build:web`
  - `npm run build:contracts`
  - `npm run build:runtime-core`

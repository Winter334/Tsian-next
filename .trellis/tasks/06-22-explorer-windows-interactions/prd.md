# 资源管理器对齐 Windows 交互：新建/复制/剪切/粘贴/快捷键

## Goal

让 `WorkspaceExplorerView.vue` 的文件操作体验对齐 Windows 资源管理器：新建不弹窗、直接创建并进入重命名；补齐复制/剪切/粘贴；加键盘快捷键。改动集中在前端视图层，不动后端 workspace 操作 API。

## Background

当前资源管理器右键菜单只有：打开/编辑/重命名/删除（条目），刷新/新建文件（空白）。新建文件走 `prompt()` 弹窗输入文件名 → 跳转编辑器 create 模式 → 输入内容 → 保存，交互繁琐且与 Windows 习惯不符。没有复制/剪切/粘贴，没有新建文件夹，没有键盘快捷键。

## Requirements

### R1 新建文件（对齐 Windows）

- 空白区右键 →「新建文件」：不弹窗、不打开编辑器。
- 直接在当前目录创建默认名 `新建文件.txt`（空内容, `text/plain`），若已存在则递增为 `新建文件(1).txt`、`新建文件(2).txt`...
- 创建后自动选中该文件并进入重命名状态，选中文件名主干（不含扩展名），用户可直接键入新名或回车确认。

### R2 新建文件夹（对齐 Windows）

- 空白区右键 →「新建文件夹」：不弹窗。
- 在当前目录创建默认名 `新文件夹`，若已存在则递增 `新文件夹(1)`、`新文件夹(2)`...
- 创建后自动进入重命名状态。
- **存储约束**：工作区是文件式存储,空文件夹无法持久化。通过写入一个 `.keep` 锚点文件（空内容）让文件夹出现在列表中。`.keep` 是目录的**持久锚点**：
  - 新建文件夹时写入 `<dir>/.keep`。
  - `.keep` 只随「删除整个目录」或「重命名/移动整个目录」消失（目录级操作,后端按前缀匹配连 `.keep` 一起处理）。
  - 删除目录内的单个文件、把单个文件移出目录,都不影响 `.keep`——后端按精确 path 删/移,不会动 `.keep`。因此「移除某个目录下所有其它文件,目录仍保留」。
  - 资源管理器列表**隐藏名为 `.keep` 的条目**,使文件夹显示为空。
  - 边界：用户通过编辑器手动删 `.keep` 且目录无其它文件 → 目录消失（高级边缘操作,可接受）。

### R3 复制 / 剪切 / 粘贴

- 条目右键菜单新增：复制、剪切、粘贴（粘贴仅在剪贴板非空时可用）。
- 空白区右键新增：粘贴（剪贴板非空时）。
- 支持文件与文件夹两者。
  - 复制文件：`read` 源 → `write` 到目标（冲突时数字递增后缀）。
  - 复制文件夹：递归 `list` + `read` 所有子文件 → 逐个 `write` 到目标前缀下（含 `.keep` 占位）。
  - 剪切文件/文件夹：`move` 源到目标（`moveWorkspacePath` 已支持目录级移动）。冲突时数字递增后缀。
- 粘贴目标始终是当前目录。
- 剪切项视觉反馈：半透明（`opacity-50`），仅在源目录可见（导航走看不到,回到源目录若未粘贴仍半透明提示），直至粘贴、取消、或切换上下文。
- 剪贴板**跨目录保留**（跨目录移动是剪切的核心用途）；**跨游戏卡 / 跨本地根（.tsian vs card）清空**（路径空间不同）；回到根目录选择界面清空（无粘贴目标）。

### R4 键盘快捷键

- `Ctrl+C`：复制当前选中项。
- `Ctrl+X`：剪切当前选中项。
- `Ctrl+V`：粘贴到当前目录。
- `Delete`：删除当前选中项（走现有确认弹窗）。
- `F2`：重命名（已有，保留）。
- `Escape`：取消重命名 / 关闭右键菜单（已有，保留并扩展）。
- 在可编辑元素（input/textarea/contenteditable）内不触发这些快捷键。
- **不做 F5**：保留浏览器原生 F5 刷新页面作为逃生通道。资源管理器已有右键「刷新」+变更事件自动刷新,不缺入口。

### R5 现有功能保持

- 重命名内联输入（已有）保留,扩展为新建后自动进入并选中文件名主干。
- 删除确认弹窗（已有）保留。
- F2 重命名（已有）保留。
- `save/` 虚拟存档槽目录不可新建/复制/剪切/粘贴/删除/重命名（已有 `canDeleteEntry` 守卫,扩展复用）。

## Constraints

- 改动集中在 `apps/platform-web/src/views/WorkspaceExplorerView.vue`。不改 `platform-host/workspace-ops.ts`、不改 `agent-runtime/workspace-operations.ts`、不改 `@tsian/contracts`。
- 遵循 spec `component-guidelines.md`：「Keep file creation and rename path decisions in resource-manager views」。
- 遵循 spec `state-management.md`：视图内 `ref`/`computed`/`watch` 管理剪贴板状态；mutation 后从 platform API 刷新。
- 遵循 spec `quality-guidelines.md`：`npm run build:web` 必须通过。
- 不引入新依赖、不改 Dexie 表、不改路由。
- 右键菜单位置计算复用现有 `contextMenuStateFromMouse` 模式,菜单项增加后调整 `menuHeight` 估算。

## Acceptance Criteria

- [ ] 空白区右键「新建文件」直接创建 `新建文件.txt`（冲突时递增），不弹窗、不跳编辑器，新建后自动进入重命名且选中文件名主干。
- [ ] 空白区右键「新建文件夹」直接创建 `新文件夹`（冲突时递增），不弹窗，新建后自动进入重命名。列表中 `.keep` 占位文件不可见。
- [ ] 条目右键菜单有「复制」「剪切」「粘贴」；空白区右键菜单有「粘贴」（剪贴板空时不可用或隐藏）。
- [ ] 复制文件 → 在同目录粘贴生成副本（数字递增后缀）；复制文件夹 → 递归复制所有子文件。
- [ ] 剪切文件/文件夹 → 粘贴后源消失；剪切期间源项半透明显示。
- [ ] `Ctrl+C/X/V`、`Delete` 快捷键可用；在输入框内不误触发。
- [ ] `save/` 虚拟存档槽不可新建/复制/剪切/粘贴/删除/重命名。
- [ ] `npm run build:web` 通过。

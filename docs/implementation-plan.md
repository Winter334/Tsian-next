# Tsian Next Implementation Plan

## 1. 文档目的

本文档记录从当前可运行原型继续推进的实施顺序。

旧版实施计划中的阶段 1 到阶段 9 已经基本完成或被当前实现覆盖，因此本文档不再重复早期骨架阶段。

当前目标是：

- 继续保持每一步都有可见效果
- 先验收已落地的新契约
- 再补平台 WebUI 的必要配置能力
- 避免在主链未稳定前提前扩展工坊、插件、复杂兼容层

## 2. 当前基线

当前项目基线如下：

- 平台 WebUI 可以启动
- 官方默认游玩前端包可以加载
- 本地 Dexie/IndexedDB 存档可用
- 已有检索 AI、正文 AI、维护 AI 最小链路
- 已有事件记忆、档案、globals、当前时间
- 已有默认测试种子
- 已有事件检索、嵌入重排和调试面板
- 新 patch 契约已经在代码中第一轮落地

已验证命令：

- `npm run build:contracts`
- `npm run build:runtime-core`
- `npm run build:web`

详细当前状态见：

- `current-state-handoff.md`

## 3. 当前实施原则

后续继续遵循：

- 先验收主链，再补配置 UI
- 先做最小可见入口，再扩完整编辑体验
- 先让用户能手动配置并跑通，再做预设、模组、工坊
- 不做旧 IndexedDB 迁移
- 不做兼容层
- 不做复杂通用 patch 系统
- 不提前做多事件并行维护

## 4. 下一阶段实施顺序

### 阶段 A：真实运行验收新契约

目标：

- 确认当前代码和新文档契约真实一致
- 确认维护 AI 能稳定输出新 patch

验收标准：

- `npm run dev:web` 后页面能正常打开
- 新建存档后使用的是 `tsian-local-v2` 新库
- 发送一轮真实游玩输入后，正文 AI 正常回复
- 控制台维护 AI 响应包含 `currentTime / globals / events / archives` 中的合理字段
- 事件更新走 `target: "active" + set`
- 档案更新走 `target + set` 或 `create`
- 快照中只有 `globals`，不再出现旧 `status`
- 档案中使用 `kind / linkedNames`，不再出现旧 `entityType / relatedEntities`

当前不要做：

- 不新增复杂 UI
- 不扩展 patch 动作
- 不补迁移
- 不重写检索策略

### 阶段 B：AI 配置最小可视化入口

目标：

- 不再依赖手改本地配置完成基础测试

验收标准：

- 平台 WebUI 能配置正文 AI、检索 AI、维护 AI 的基础参数
- 配置能保存到浏览器本地
- 发送消息时实际读取该配置
- 配置错误能在页面或控制台明确暴露

当前不要做：

- 不做多供应商高级编排
- 不做复杂密钥管理
- 不做云端账号体系

### 阶段 C：提示词预设最小入口

目标：

- 让三类核心 AI 的提示词从代码常量逐步迁出

验收标准：

- 平台 WebUI 能查看当前检索 AI、正文 AI、维护 AI 的提示词片段
- 能编辑并保存最小提示词覆盖内容
- 新一轮对话能使用修改后的提示词
- 默认提示词仍可恢复

当前不要做：

- 不做 SillyTavern 完整导入
- 不做预设市场
- 不做复杂变量系统

### 阶段 D：运行时数据查看与调试增强

目标：

- 让记忆系统调试更直接，方便继续验证设计

验收标准：

- 平台或官方前端能清楚查看当前快照
- 能查看完整事件列表
- 能查看完整档案列表
- 能查看 globals
- 能查看最近一次维护 patch 原文

当前不要做：

- 不做编辑器型后台
- 不做版本回滚
- 不做审计系统

### 阶段 E：模组最小装载

目标：

- 让“模组作为内容产品”开始成立

验收标准：

- 至少存在一个内置模组定义
- 新建存档时可以选择模组
- 不同模组可以提供不同初始档案、初始事件、globals 和前端包配置
- 模组内容与运行中存档区分清楚

当前不要做：

- 不做上传下载
- 不做工坊
- 不做版本兼容
- 不做复杂依赖和冲突处理

### 阶段 F：前端包桥 API 收敛

目标：

- 固定游玩前端包与平台 WebUI 的最小通信边界

验收标准：

- 前端包能读取快照、历史、事件、档案、检索调试信息
- 前端包能发送玩家输入
- 内部 AI 调度、上下文组装、patch 应用仍不暴露给前端包
- 官方默认前端包只依赖这套桥 API

当前不要做：

- 不做第三方包沙箱
- 不做权限系统
- 不做远程包加载

## 5. 推荐当前下一步

当前最建议先做：

`阶段 A：真实运行验收新契约。`

原因：

- 新契约刚落地，必须先验证维护 AI 是否能稳定按新格式输出
- 如果新契约实际运行不稳，继续做配置 UI 会放大问题
- 这一步不引入新系统，只验证当前主链

阶段 A 通过后，再进入：

`阶段 B：AI 配置最小可视化入口。`
